                          COMO USAR ESTA DOCUMENTACIÓN
================================================================================

SCENARIO 1: Testing Antes de Release
─────────────────────────────────────
1. Leer QA_CHECKLIST.md
2. Ejecutar items de checklist
3. Referirse a TEST_SUITE.md para detalles de cada test
4. Verificar navegadores en BROWSER_COMPATIBILITY.md
5. Documentar resultados
6. Firmar checklist cuando todo PASS

SCENARIO 2: Debugging de Bug
────────────────────────────
1. Abrir KNOWN_ISSUES.md
2. ¿Ya está reportado?
3. Si sí → Usar workaround
4. Si no → Reportar nuevo issue
5. Verificar navegadores afectados en BROWSER_COMPATIBILITY.md
6. Esperar fix o usar workaround temporal

SCENARIO 3: Optimización de Performance
──────────────────────────────────────
1. Leer PERFORMANCE_BENCHMARKS.md
2. Ejecutar mediciones (Lighthouse, DevTools)
3. Comparar con targets
4. Implementar optimización
5. Medir nuevamente
6. Documentar cambios

SCENARIO 4: Deployment a Producción
───────────────────────────────────
1. 48h antes: Leer DEPLOYMENT_CHECKLIST.md
2. Ejecutar pre-deployment items
3. Ejecutar QA_CHECKLIST.md (final verification)
4. Revisar PERFORMANCE_BENCHMARKS.md (debe cumplir targets)
5. Día del deploy: Seguir DEPLOYMENT_CHECKLIST.md paso a paso
6. Post-deploy: Ejecutar verificación inmediata
7. 24h después: Ejecutar verificación extendida

SCENARIO 5: Onboarding de QA Engineer
─────────────────────────────────────
1. Leer esta página (5 min)
2. Leer BROWSER_COMPATIBILITY.md (10 min)
3. Leer QA_CHECKLIST.md (15 min)
4. Leer TEST_SUITE.md (20 min)
5. Ejecutar 5 tests de TEST_SUITE (15 min)
6. Reportar un bug dummy en KNOWN_ISSUES.md (10 min)
7. Ready para empezar! ✅
